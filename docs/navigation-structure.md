# Структура навігації Berry POS

## Огляд

Цей документ описує структуру навігації системи Berry POS, включаючи компоненти, їх взаємодію та правила оновлення.

## Основні компоненти

### 1. DashboardLayout

**Файл**: `src/components/layout/DashboardLayout.tsx`
**Опис**: Основний компонент макету, який забезпечує навігацію для всіх backend сторінок.

#### Структура:

```
DashboardLayout
├── Sidebar (Фіксований, зліва)
│   ├── Header (з логотипом та кнопкою згортання)
│   ├── Navigation (пункти меню)
│   └── User Profile (профіль користувача)
└── Main Content
    ├── Top Header (фіксований, зверху)
    └── Page Content (контент сторінки)
```

### 2. Sidebar

**Позиція**: Фіксована зліва
**Z-index**: 50
**Висота**: 80px (h-20)
**Ширина**:

- Розгорнутий: 256px (w-64)
- Згорнутий: 64px (w-16)

**Тіні**:

- Розгорнутий: `shadow-lg` (стандартна тінь)
- Згорнутий: `shadow-[2px_0_8px_rgba(0,0,0,0.1)]` (тонка тінь справа)

#### Header Sidebar:

- Логотип "Berry POS" (тільки в розгорнутому стані)
- Кнопка згортання/розгортання (ChevronLeft/ChevronRight)
- Висота: 80px
- Padding: 16px (p-4)

#### Navigation:

- Пункти меню з іконками та описом
- Активний стан з синім кольором та правою рамкою
- Прокрутка при необхідності
- Висота: calc(100vh - 140px)

#### User Profile:

- Аватар користувача (перша літера email)
- Email користувача (тільки в розгорнутому стані)
- Кнопка виходу з системи
- Позиція: абсолютна внизу Sidebar

### 3. Top Header

**Позиція**: Фіксована зверху
**Z-index**: 40
**Висота**: 80px (h-20)
**Ширина**: Автоматично адаптується до стану Sidebar

#### Структура:

- Кнопка меню (тільки на мобільних пристроях)
- Назва поточної сторінки
- Опис сторінки
- Індикатор бізнесу (праворуч)

## Пункти навігації

### Поточні пункти:

1. **Dashboard** (`/dashboard`)
   - Іконка: LayoutDashboard
   - Опис: Головна панель

2. **POS Система** (`/pos`)
   - Іконка: ShoppingCart
   - Опис: Швидке створення замовлень
   - **Особливості**: Повноекранний інтерфейс без DashboardLayout

3. **Меню** (`/menu`)
   - Іконка: Menu
   - Опис: Управління меню
   - **Підменю**:
     - **Огляд** (`/menu`) - Загальний огляд меню
     - **Категорії** (`/menu/categories`) - Управління категоріями
     - **Продукти** (`/menu/products`) - Управління продуктами

4. **Замовлення** (`/orders`)
   - Іконка: Receipt
   - Опис: Управління замовленнями

5. **Клієнти** (`/customers`)
   - Іконка: Users
   - Опис: Управління клієнтами

6. **Аналітика** (`/analytics`)
   - Іконка: BarChart3
   - Опис: Аналітика та звіти

7. **Налаштування** (`/settings`)
   - Іконка: Settings
   - Опис: Налаштування бізнесу

## Особливості навігації

### POS Інтерфейс

**Сторінка**: `/pos`
**Компонент**: `POSInterface`
**Макет**: Без DashboardLayout (повноекранний)

**Функції**:
- Швидкий пошук товарів
- Корзина замовлення
- Вибір клієнта
- Оформлення замовлення
- Реальний час оновлення

**Навігація**:
- Повноекранний режим для зручності роботи
- Швидкий доступ через головне меню
- Можливість повернення до dashboard

### Управління замовленнями

**Сторінка**: `/orders`
**Компонент**: `OrderManagement`
**Макет**: З DashboardLayout

**Функції**:
- Перегляд всіх замовлень
- Фільтрація по статусу, клієнту, даті
- Оновлення статусів
- Детальний перегляд
- Експорт даних

### Управління клієнтами

**Сторінка**: `/customers`
**Компонент**: `CustomerManagement`
**Макет**: З DashboardLayout

**Функції**:
- CRUD операції з клієнтами
- Програма лояльності
- Пошук та фільтрація
- Експорт бази клієнтів

## Правила оновлення

### Додавання нових пунктів меню:

1. **Створення компонента** в `src/components/business/`
2. **Створення сторінки** в `src/app/[route]/page.tsx`
3. **Оновлення навігації** в `DashboardLayout.tsx`
4. **Оновлення документації** в `navigation-structure.md`

### Структура компонентів:

```
src/
├── app/
│   ├── pos/
│   │   └── page.tsx          # POS інтерфейс
│   ├── orders/
│   │   └── page.tsx          # Управління замовленнями
│   ├── customers/
│   │   └── page.tsx          # Управління клієнтами
│   └── menu/
│       ├── page.tsx          # Огляд меню
│       ├── categories/
│       │   └── page.tsx      # Категорії
│       └── products/
│           └── page.tsx      # Продукти
└── components/
    └── business/
        ├── POSInterface.tsx      # POS інтерфейс
        ├── OrderManagement.tsx   # Управління замовленнями
        ├── CustomerManagement.tsx # Управління клієнтами
        ├── MenuManagement.tsx     # Управління меню
        └── ProductManagement.tsx  # Управління продуктами
```

```

/menu               # Управління меню
/menu/categories    # Категорії
/menu/products      # Продукти
/orders             # Замовлення
/customers          # Клієнти
/analytics          # Аналітика
/settings           # Налаштування
```

### Responsive дизайн:

- **Desktop**: Повна навігація з розгорнутим sidebar
- **Tablet**: Адаптивна навігація з можливістю згортання
- **Mobile**: Мобільна навігація з гамбургер меню

## Висновок

Структура навігації Berry POS забезпечує:

1. **Зручність використання** - інтуїтивна навігація
2. **Швидкість роботи** - швидкий доступ до всіх функцій
3. **Масштабованість** - легко додавати нові розділи
4. **Консистентність** - єдиний стиль для всіх сторінок
5. **Responsive дизайн** - робота на всіх пристроях

Ця структура дозволяє користувачам ефективно працювати з системою та швидко знаходити потрібні функції.
